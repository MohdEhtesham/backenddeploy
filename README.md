# Aabroo Backend

Production-ready Node.js + Express + MongoDB API powering the Aabroo real-estate marketplace mobile app (consumer + seller flows).

## Stack
- **Node.js** 18+
- **Express** 4
- **MongoDB** (Atlas free tier recommended)
- **Mongoose** ODM
- **JWT** auth + bcrypt password hashing
- **Cloudinary** for image uploads (with multer)
- **Zod** for request validation
- Helmet + CORS + rate-limit security middleware

## Quick start (local)

1. **Install Node 18+** and have a MongoDB connection string.
2. Get a free **MongoDB Atlas** cluster: https://www.mongodb.com/cloud/atlas (M0 free tier)
3. Get free **Cloudinary** credentials: https://console.cloudinary.com/

```bash
cd backend
cp .env.example .env
# Edit .env and fill MONGODB_URI, JWT_SECRET, CLOUDINARY_*

npm install
npm run seed        # populate curated catalog + demo user
npm run dev         # http://localhost:4000
```

Test it:
```bash
curl http://localhost:4000/health
```

Demo credentials after seed: `demo@aabroo.com` / `Aabroo@123`

## Deploy to Render.com (free tier)

1. Push this repo to GitHub.
2. Go to https://dashboard.render.com → **New +** → **Blueprint** → connect GitHub repo.
3. Render reads `render.yaml` and creates the web service.
4. After deploy, set the secrets in the Render dashboard:
   - `MONGODB_URI` — your Atlas connection string
   - `CLOUDINARY_CLOUD_NAME`, `CLOUDINARY_API_KEY`, `CLOUDINARY_API_SECRET`
   - (`JWT_SECRET` is auto-generated by Render)
5. **Manual Deploy → Deploy latest commit**.
6. Copy the public URL — that goes into your mobile app's `API_BASE_URL`.

After first deploy, run the seed once via Render Shell:
```bash
npm run seed
```

## API Reference

Base URL: `http://localhost:4000/api`

All authenticated endpoints require `Authorization: Bearer <token>`.

Standard response envelope:
```json
{ "success": true, "data": {...}, "message": "optional" }
```

| Method | Path | Auth | Body |
|---|---|---|---|
| POST | `/auth/signup` | – | `fullName, email, phone, password, role?` |
| POST | `/auth/login` | – | `identifier, password, role?` |
| POST | `/auth/otp/send` | – | `phone` |
| POST | `/auth/otp/verify` | – | `phone, otp, role?` (mock OTP = `1234`) |
| POST | `/auth/forgot-password` | – | `identifier` |

## Frontend integration

The mobile app reads `API_BASE_URL` from `src/config/env.ts`. Set it to your Render URL after deploy:

```ts
export const API_BASE_URL = 'https://aabroo-api.onrender.com';
export const USE_MOCK = false; // toggle to false to hit real API
```

When `USE_MOCK=true`, services use the in-app mock data (no network calls). Useful for offline dev and demos.
